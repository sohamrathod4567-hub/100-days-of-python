# 🐞 Day 13 - Debugging in Python

## 📖 Overview

Today was all about **Debugging**. Instead of building a project, I learned how to identify, understand, and fix bugs in Python programs.

I explored different debugging techniques, including reading error messages, tracing code execution, using print statements, and taking advantage of the **built-in debugger** available in modern IDEs. I also learned how **breakpoints** can make debugging much easier by allowing me to inspect my code step by step.

---

## 🧠 Concepts Practiced

* Understanding what bugs are and why they occur
* Reading and interpreting Python error messages
* Using `print()` statements for debugging
* Finding and fixing logical errors
* Using the built-in debugger in the IDE
* Setting and removing breakpoints
* Stepping through code line by line
* Inspecting variable values during execution
* Understanding program flow while debugging

## 💻 Example

Example of using a breakpoint while debugging:

```python
number = int(input("Enter a Number: "))

if number % 2 == 0:
    print("Even")
else:
    print("Odd")
```

Place a **breakpoint** on the `if` statement, run the debugger, and inspect the value of `number` before the condition is evaluated.

## 📚 What I Learned

* Debugging is a skill that every programmer needs.
* Reading error messages carefully often points directly to the problem.
* Breakpoints make it easier to pause execution and inspect variables.
* The IDE's built-in debugger is much more effective than relying only on `print()` statements.
* Stepping through code helps understand exactly how a program executes.
* Fixing bugs becomes much faster when using the right debugging tools.
